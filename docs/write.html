文档编写

环境搭建

wzero平台基于 Hugo 开发，需要您预先在本地安装最新版Hugo，Hugo官方提供了非常详细的各平台（Mac OS、Windows、Linux等）安装方式，您可以根据官方教程自行安装。

注意：wzero文档使用Git作为版本管理，请务必先确保本机已经按照好Git，如果没有安装，请去Git官网下载安装。

不过，推荐使用我们提供的脚本来搭建文档编写环境，更加方便：

1.下载对应系统安装脚本

  • Windows
  • Linux
  • Mac OS

2. 运行脚本创建项目

  • Windows下右键以管理员身份运行脚本
  • Linux或Mac下终端运行 bash eco-docs.sh 命令

注意：输入项目名建议使用项目英文简称，将用于创建项目文件夹

3.浏览器访问项目文档

创建项目后，会自动运行项目文档，打开浏览器访问 http://localhost:1313 查看文档。

编写工具

可以使用任何您喜欢的文本编辑器来编写Markdown，如果您还不太了解Markdown，推荐您使用 Visual Studio Code，它是一个跨平台（支持Mac OS、Windows、Linux）的文本编辑器，对于Markdown也提供了非常友好的支持。

快速上手

下面以Mac平台为例，演示文档创建及编写的整个过程：

Step 1：创建文档

终端运行 bash eco-docs.sh

会提示 please input your project name：，这里输入 eco-doc-demo，表示wzero文档示例教程。

接下来会提示 Preview now? 是否预览，输入 y 选择直接预览。

浏览器访问 http://localhost:1313，出现如下界面，则表示示例文档创建成功。

Step 2：修改文档名称

注意：用喜爱的编辑器导入整个示例文档项目，编辑文档内容。

修改 /data/menu/menu.toml，修改内容如下：

name = "wzero示例"
description = "这是wzero文档的示例"

menu.toml是一个toml格式的默认主菜单配置文件，默认已经配置了 name 和 description属性，其中：

  • name：表示文档名称，显示在文档左上角（搜索框上面）

  • description：表示文档描述，建议填写一段简短的描述，不显示，对搜索引擎友好

Step 3：修改首页内容

修改 /content/index.md 文件，修改内容如下：

+++
title = ""
description = "这是wzero文档示例教程"
date = "2017-02-01"
draft = false
url = "/"
+++

# wzero文档示例教程

这是一个wzero文档示例教程，我们将根据该文档一步步尝试着编写教程。
```toml
title = ""
description = "这是wzero文档示例教程"
date = "2017-02-01"
draft = false
url = "/"
/```

`index.md`的头部是toml格式的文件属性，其中：

- title：表示文档title中的显示名称，可用于拼接title，首页index建议不写，否则会导致首页title名称拼接冗余
- description：表示文档首页的description描述，建议填写一段简短的描述，不显示，对搜索引擎友好
- date：表示文档的创建时间，可填写实际时间
- draft：默认为 `false`，表示不是草稿文档，草稿文档最终不会生成
- url：默认为 `/`，表示根目录访问，不要修改

Good，你已经了解了index.md文件的构成，下面我们开始继续编写吧！

完善现有文档

上面讲解了怎么从无到有的创建一个示例文档，并上传至Git托管平台。通常，文档需要团队多人协同、长期持续编写完善，那么怎么基于现有的文档仓库多人协同编写呢？

Step 1：克隆文档仓库

依然以上文的 eco-doc-demo 文档为例，运行以下命令克隆文档仓库：

git clone --recursive https://github.com/purman/purman.github.io.git

注意：git clone 时别忘了添加 --recursive 选项，会自动去拉取主题样式，没有主题会导致文档无法正常访问，一片空白。

Step 2：文档编写与提交

多人协同开发，建议参考Git Flow工作方式，团队成员可以基于自己的独立分支编写文档，然后合并至master分支，如果涉及到多个文档版本，可以利用Git的branch或者tag来实现，具体工作方式视团队情况确定。

常用命令

Hugo提供了很多命令，可以通过 hugo --help 查看，wzero文档平台本着简单、易用的原则，只需要掌握以下几个常用命令即可：

注意：为执行以下命令，请确保已经进入文档目录，如 cd eco-doc-demo

  • hugo server：运行文档平台，运行成功后，默认可以通过 http://localhost:1313访问，如果端口已被占用，会选择一个未占用的其他端口。
  • hugo new：添加新文档到 content 目录下
    • 如，hugo new test1.md，会在content 目录下创建 test1.md
    • 如，hugo new menu1/test1.md，会在 content/menu1 目录下创建 test1.md
    • 如，hugo new menu1/menu2/test1.md，会在 content/menu1/menu2 目录下创建 test1.md
    • 注意：建议目录名跟菜单名保持一致
  • hugo：默认的 hugo 命令，会将文档生成静态HTML页面到 public 目录下，可以将 public 目录进行文档分发，直接点击 public/index.html即可访问
  • 删除文档，删除文档直接删除 content 下的相应的 Markdown 文件即可

以上，一般只需要掌握以上几个命令，即可熟练的使用wzero文档平台编写文档。

编写技巧

目录结构

.
├── archetypes
│   └── default.md  // 新文档默认格式，无需修改
├── config.toml  // 全局配置，无需修改
├── .gitignore  // .gitignore文件，按需修改
├── .gitmodules  // git子模块，引用wzero文档主题，无需修改
├── content  // 文档主目录，所有文档都写在这里
│   └── index.md  // 默认文档首页，可按需修改首页内容
├── data
│   └── menu  // 默认菜单目录，用于编写文档菜单
│       └── menu.toml  // 默认主菜单文件，按需修改菜单内容
├── layouts
├── static
└── themes
    └── hep-docs-theme  // git引入的wzero文档主题子模块，无需修改

作为文档维护者，只需要关注在 /content 目录中编写文档内容以及在 /data/menu目录中编写文档菜单即可，其他无需关注。

图片资源

文档中经常会引入图片资源，建议在所属文档的当前同级目录下创建 images 文件夹，将图片资源存到该目录下，再文档中以下面方式引入：

![image_xxx](./images/images_xxx.xxx)

菜单

如果文档菜单很少，建议将菜单放到同一个 menu.toml 文件中配置；如果菜单过多，可以分拆成多个子菜单配置文件放到 menu 目录下。

Git仓库权限

wzero内部使用Gitlab作为Git托管平台，wzero文档平台支持接入公有（public）及私有（private）仓库，为保证仓库安全性，针对私有（private）仓库不要求提供访问秘钥，只需要在 Gitlab中将 wzero（用户名：eco） 用户添加到相应Git仓库，并授予 Reporter 权限即可。

文档内搜索

wzero文档平台提供了文档内搜索功能，对于发布到wzero文档平台的文档会自动触发生成文档索引，提供文档内全文搜索功能。

注意：暂时不提供本地搜索配置方式，后续会支持该功能。

更新主题

wzero文档提供了默认主题，位于 themes/wzero-theme，以 git submodule 的方式引入，该主题会一直持续更新、完善，为了获得更好的用户体验，建议您不定期更新最新主题。

更新方式：

  1. 进入主题目录：cd themes/wzero-theme
  2. 切换至master分支：git checkout master
  3. 拉取最新主题：git pull

根据以上三步拉取最新主题以后，返回文档根目录，通过 hugo server 运行文档，浏览器访问，即可体验最新主题功能。

注意：如果文档托管到wzero文档平台，那么无需手工更新主题，平台会自动以最新主题发布文档。所以，更新主题主要针对本地编写及预览用。
